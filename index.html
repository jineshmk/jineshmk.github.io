Constrained Objects

Constrained objects have been proposed as a modeling framework for complex systems that consists of an assembly of interconnected, interdependent components with two essential characteristics: (i) They are compositional, i.e., a complex structure is built (recursively) of smaller structures. (ii) Laws or rules regulate the behavior of an individual component by itself and its relation to other components. This programming methodology is applicable in the various system from physical and engineering domains. The constrained objects language utilizes the object-oriented programming concepts such as classes, inheritance, and aggregation for structural modeling and declarative constraints for behavioral modeling. The emergent behavior of a collection of constrained objects is determined by the process of constraint solving over the attributes of objects. Read More

Temporal Constrained Objects

Temporal constrained objects added a temporal extension to constrained object framework, which facilitates the declarative specification of time-dependent behavior in complex dynamic systems. The language TCOB implements this concept using series variable, which record the dynamic changes of the system, and the temporal constraints are placed over the consecutive values of this variable. The language introduces a more general approach to temporal constraints through the introduction of metric temporal operators which are metric variants of the classic F and G operators of linear temporal logic (LTL) . The execution model for temporal constrained objects language involves a time-based simulation along with constraint solving at each time-step. Read More
